## Методология расчёта

### 1. Коэффициент LTV (% от D180)

Линейная интерполяция между опорными точками (weighted avg из 20 mature когорт):

| День | % от D180 | Коэффициент |
|------|-----------|-------------|
| D1 | 19.85% | 0.1985 |
| D4 | 28.54% | 0.2854 |
| D7 | 30.35% | 0.3035 |
| D14 | 36.97% | 0.3697 |
| D30 | 47.67% | 0.4767 |
| D60 | 64.46% | 0.6446 |
| D90 | 75.75% | 0.7575 |
| D120 | 85.00% | 0.8500 |
| D180 | 100% | 1.0000 |

**Формула для дня X между Day_low и Day_high:**
```
coef(X) = (pct_low + (X - low) / (high - low) × (pct_high - pct_low)) / 100
```

### 2. Предикт D180

```
D180_predict = текущий_ROAS / coef(текущий_день)
```

### 3. День окупаемости

**Если D180_predict ≥ 100%:**
```
Найти день Y где coef(Y) = текущий_ROAS / 100
```

**Если D180_predict < 100%:**
```
breakeven_day = 180 + (100 - D180_predict) / 0.15
```

Рост после D180: **0.15%** в день (среднее из mature когорт)

## Как проверить точность

После того как когорта достигнет D180:

```sql
-- Сравнить предикт с фактом
SELECT
  month,
  d180_predict as predicted,
  roas_d180 as actual,
  ABS(d180_predict - roas_d180) as error
FROM cohort_predictions cp
JOIN (
  SELECT cohort_month, SUM(roas_d180_pct * spend) / SUM(spend) as roas_d180
  FROM cohort_roas
  GROUP BY cohort_month
) cr ON cp.month = TO_CHAR(cr.cohort_month, 'YYYY-MM')
WHERE days >= 180;
```
